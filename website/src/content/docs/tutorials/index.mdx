Tutorials are the project-shaped side of the docs. Where guides cover one task and concepts cover one piece of the model, a tutorial walks you through a complete shape: a runtime, a set of plugins, an end-to-end story.

Each one is paired with a runnable example in the [`example/`](https://github.com/SaadArdati/plugin_kit/tree/main/example) directory. Read the page, run the code, swap things around. The example is the source of truth; the tutorial is the map.
