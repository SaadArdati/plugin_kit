Plugin Kit is the part of the application most teams build by accident. It packages real feature lifecycles,
replaceable services, isolated sessions, and typed events into one runtime with explicit rules. You write plugins;
it does the choreography.

## What it unlocks

<CardGrid stagger>
	<Card title="Swap features, not screens" icon="random">
		A model selector, search box, or editor tool targets a slot instead of a
		concrete service. The UI stays put while plugins compete underneath it.
		Override stops being a code smell.
	</Card>
	<Card title="Open two sessions safely" icon="approve-check-circle">
		Two chats, documents, or workspaces get separate registries, buses, and
		plugin state by default. No mystery leaks because one tab forgot to clean
		up after itself.
	</Card>
	<Card title="Add a 'hooks' pipeline" icon="pencil">
		Plugins inspect an outgoing action, inject context, enrich metadata, or
		stop it entirely. The button that started it does not need to know who
		helped or interfered.
	</Card>
	<Card title="Toggle features live" icon="setting">
		A customization dialog enables, disables, and reprioritizes plugins at
		runtime. The runtime detaches losers, attaches winners, and keeps the
		session coherent across the swap.
	</Card>
</CardGrid>

## Four primitives, composed

<CardGrid>
	<Card title="Plugins" icon="puzzle">
		Lifecycle-aware feature units. Global or session scope, with `register`,
		`attach`, `detach`, and live settings reconciliation as first-class hooks.
	</Card>
	<Card title="Services" icon="seti:db">
		A priority-based registry where multiple plugins can claim the same slot.
		Higher priority wins, the loser stays reachable as a fallback.
	</Card>
	<Card title="Events" icon="rocket">
		A typed, priority-ordered bus for fire-and-forget and request/response.
		Handlers run in order, so plugins can mutate, observe, or short-circuit.
	</Card>
	<Card title="Sessions" icon="document">
		Isolated execution scopes. Each gets its own registry, bus, and context.
		Open ten and close one without touching the others.
	</Card>
</CardGrid>
